# 查图

一个面向 Tampermonkey 的图片悬浮预览脚本。

当鼠标悬停在网页图片上时，插件会在鼠标反方向显示一张更大的预览图，并在预览框外展示图片真实尺寸，方便快速看图、查图和确认分辨率。

## 项目特点

- 悬停图片即可快速放大预览
- 预览优先显示在鼠标反方向，尽量减少遮挡
- 根据窗口大小自动缩放预览，避免超出视口
- 在预览框外显示真实分辨率，例如 `1920 x 1080 px`
- 自动过滤过小图片，减少误触发
- 自动排除常见图标、表情、贴纸和 reaction 小图
- 自动排除常见头像类图片，降低社交页面干扰
- 对快速移动、滚动、缩放、失焦等场景做了稳定性优化

## 适用场景

- 浏览图片站、素材站、电商站时快速放大查看图片
- 在网页中快速确认图片分辨率
- 浏览包含大量缩略图的页面时提升查看效率
- 减少右键打开图片或新开标签页查看的操作

## 功能预览

插件的核心交互如下：

1. 鼠标移动到网页中的图片上
2. 稍作停留后显示放大预览
3. 预览框自动选择更合适的位置
4. 预览框下方显示图片真实尺寸
5. 鼠标移开后自动隐藏预览

## 安装方法

### 1. 安装 Tampermonkey

先在浏览器中安装油猴扩展：

- Chrome / Edge: [Tampermonkey](https://www.tampermonkey.net/)
- Firefox: [Tampermonkey](https://www.tampermonkey.net/)

### 2. 导入脚本

仓库中的主脚本文件为：

`查图-1.2.0.js`

安装方式任选其一：

1. 打开 Tampermonkey 控制面板
2. 新建脚本
3. 用本项目脚本内容替换默认模板
4. 保存并启用

或者：

1. 直接打开 `.js` 脚本文件
2. 由 Tampermonkey 接管安装
3. 点击安装并启用

## 使用方法

安装完成后无需额外配置，打开任意包含图片的网页即可使用。

### 基本使用

1. 打开任意包含图片的网页
2. 将鼠标移动到图片上
3. 稍作停留后，插件会自动显示放大预览
4. 预览框下方会显示该图片的真实尺寸
5. 鼠标移开图片后，预览会自动隐藏

### 显示逻辑

- 如果鼠标位于图片左侧，预览通常显示在图片右边
- 如果鼠标位于图片右侧，预览通常显示在图片左边
- 如果页面空间不足，插件会自动调整位置，尽量保证预览完整显示
- 预览大小会根据浏览器窗口尺寸动态缩放

## 过滤规则

为了减少干扰，插件默认不会对以下类型的图片弹出预览：

- 尺寸过小的图片
- 常见 UI 图标
- 表情、贴纸、reaction 小图
- 常见头像、个人资料头像、圆角头像

这样可以显著减少评论区、聊天区、导航栏和社交页面中的误触发。

## 稳定性优化

当前版本重点优化了稳定性，主要包括：

- 避免重复绑定事件，减少长时间使用后的卡顿和异常触发
- 统一管理显示与隐藏状态，降低闪烁和抖动
- 使用单一预览层复用，减少频繁创建 DOM 带来的性能损耗
- 在窗口缩放、滚动、页面失焦等情况下自动修正或隐藏预览
- 对图片加载失败场景做了容错处理

## 兼容说明

- 脚本默认匹配所有网页：`*://*/*`
- 适用于大多数网页中的标准 `img` 元素
- 某些使用背景图、Canvas、Shadow DOM 或高度自定义组件的网站，可能无法完全覆盖

## FAQ

### 1. 为什么有些图片没有触发预览？

可能原因包括：

- 图片尺寸太小，被过滤规则排除了
- 图片本身是头像、图标、表情等低优先级目标
- 页面使用的是背景图或特殊渲染方式，而不是标准 `img`

### 2. 为什么某些网站效果不一致？

不同网站的前端结构差异很大。这个脚本主要针对常见网页图片元素优化，极少数高度定制页面可能需要单独兼容。

### 3. 可以自定义过滤规则吗？

目前默认规则已经内置在脚本中。如果后续需要，可以继续扩展为可配置版本，例如：

- 自定义最小图片尺寸
- 自定义头像过滤开关
- 自定义站点黑名单 / 白名单

### 4. 可以支持背景图元素吗？

当前版本主要处理 `img` 元素。背景图支持可以作为后续增强方向加入。
